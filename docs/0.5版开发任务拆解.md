# 0.5版开发任务拆解：稳定性与长停顿体验

## 1. 目标与范围

版本目标：
1. 解决“长停顿后丢后半段文本”。
2. 解决“连续使用后偶发无法识别”。
3. 强化状态可见性与故障可诊断性。

范围依据：
- /Users/zzw/Documents/语音识别/docs/0.4版使用遇到的问题.md
- /Users/zzw/Documents/语音识别/docs/0.5版迭代计划.md

## 2. 任务分解（按优先级）

## 3. 实施顺序（推荐）

1. **第一阶段：防丢字主链路**（T-01~T-05）
2. **第二阶段：稳定性自愈链路**（T-06~T-09）
3. **第三阶段：增强体验与参数化**（T-10~T-12）

## 4. 每项任务完成定义（DoD）

每个任务必须同时满足：
1. 有对应的单元测试或集成测试。
2. 关键路径有结构化日志字段（至少含 `session_id`、`state`、`error_code`）。
3. 不引入新的未处理状态（`ERROR` 后可回 `IDLE`）。
4. 在 `tail -f ~/Library/Logs/ASR-Assistant.log` 可观察到关键步骤。

## 5. 里程碑与验收门禁

## 6. 风险与应对

1. **ASR 服务尾包时序不稳定**：
   - 应对：drain + 超时双保险，超时后走 partial 回退。
2. **重连引发状态抖动**：
   - 应对：恢复动作串行化，限制恢复重入。
3. **参数过多影响易用性**：
   - 应对：仅开放 3 个参数，默认值覆盖 80% 场景。

## 7. 发布前检查单

1. 新状态机路径单测覆盖 100%。
2. 长停顿回归 30 次完整率 100%。
3. 连续压力回归（200 短按 + 50 长按）成功率 >= 99%。
4. 故障快照可定位问题（至少可还原失败前后 10 秒事件）。
5. 灰度开关可一键回退 0.4 收尾逻辑。
