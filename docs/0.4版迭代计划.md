# 0.4版迭代计划：LLM 润色与历史记录增强

## 1. 背景与目标
在 0.3.0 版本实现了基础的“按住说话、松开上屏”的实时语音识别后，0.4.0 版本将引入大型语言模型（LLM）对识别结果进行智能润色，以提升长段落表达的逻辑性和清晰度；同时引入历史记录功能，方便随时找回历史输入。

## 2. 核心功能点

### 2.1 双快捷键支持
*   **基础模式**：保留现有的快捷键（默认 `Option`），逻辑不变（即按住录音，松开直接粘贴原始识别文本）。
*   **LLM 润色模式**：新增第二个快捷键（默认 `右 Option`），按住该键录音，松开发送给大模型进行润色后，再将润色后的文本粘贴到目标输入框。
*   **按键配对规则**：哪个键按下启动会话，就只能由同一个键的松开事件结束会话，避免两个快捷键交叉触发。

### 2.2 LLM 智能润色
*   **选用模型**：阿里云通义千问 `qwen3.5-plus` 兼容模式。
*   **实现方式**：使用 `openai` Python 客户端，指向 `https://dashscope.aliyuncs.com/compatible-mode/v1`。
*   **润色逻辑**：内置 Prompt，要求“确保意图不变，逻辑清晰（如按 1、2、3 列出点）”。
*   **交互体验**：识别完成后，状态变更为“生成中 / 润色中”，前端悬浮窗及状态栏图标提示用户等待片段，生成结束后自动采用黏贴的形式输出到目标输入框。
*   **失败回退策略**：若 LLM 调用失败，自动回退粘贴原始识别文本，并提示错误，不中断输入流。

### 2.3 Markdown 历史日志
*   **存储格式**：按天生成 Markdown 文件，每天固定一个文件（例如 `chat2word_2026-02-22.md`）。
*   **内部格式设计**：
    每个记录作为独立的一节，包含：
    *   **时间戳标题**（如 `### 14:30:15`）
    *   **🎯 识别原文**：直接语音识别出的基础文本。
    *   **✨ 润色结果**（可选）：使用 LLM 功能时，额外附上的通过大模型润色后的文本。
*   **多维度记录**：完整保留“原始识别文本”以及“LLM 润色后的文本”（如果使用了 LLM 模式）。
*   **防丢机制**：无论粘贴是否成功，都会自动追加到本地文件。

## 3. 架构改造与模块划分

1.  **Hotkey 适配器扩展 (`hotkey.py`)**：修改 `GlobalHotkeyAdapter`，使其能同时注册和区分两个不同的监听键（例如传入 `key_primary` 和 `key_secondary`），并在回调中告知是哪一个键触发。
2.  **Session 控制器状态机 (`session_controller.py`)**：扩展状态枚举，加入 `LLM_PROCESSING` 状态。识别出 `FINAL` 结果后，判断依据的触发键决定是直接 `PASTING` 还是进入 `LLM_PROCESSING`。
3.  **LLM 处理器 (`llm_adapter.py` 新增)**：封装基于 `openai` 客户端对 `qwen3.5-plus` 的调用，异步或在后台线程执行生成请求，防止阻塞主线程与 UI 事件循环。
4.  **日志系统 (`history_logger.py` 新增)**：专门负责格式化文本并按天追加写 Markdown 文件，需注意线程安全或独占写机制。
5.  **配置与 UI (`config.py`, `main.py`)**：支持设置第二热键，UI 状态及托盘菜单需添加对 LLM 生成状态及错误异常的捕获显示。
