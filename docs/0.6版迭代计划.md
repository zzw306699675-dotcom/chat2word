# 0.6版迭代计划：上屏一致性与过程可见性优化

## 1. 背景
0.5 版本已经完成了基础稳定性治理（收尾状态机、文本聚合、自恢复、健康日志），但真实使用反馈显示：
1. 菜单语言仍为英文，日常使用不够直观。
2. 语音中间停顿 2-3 秒时，悬浮字幕会中断消失，影响思路连续性。
3. 在部分机器上出现“松键后只上屏前几个字，下一次按键才出现剩余内容”。
4. 缺少“当前光标是否在可输入框”的实时提示，容易说完才发现无法上屏。
5. 存在“首段先上屏、尾段延迟到下一次会话”的分段错位问题。
6. 收尾阶段偶发 4-5 秒延迟，影响输入节奏。

0.6 的核心目标是：**一次会话内“所见即所得、一次上屏完整、状态全程可感知”**。

## 2. 0.6 版本目标（体验指标）
1. **过程不断流**：停顿期间字幕不丢失、不闪断。
2. **上屏强一致**：一次松键只触发一次完整上屏，不把尾段拖到下一次会话。
3. **输入前可预判**：开始说话前或说话中可看到“是否可上屏”。
4. **收尾更快**：绝大多数场景在 1.5 秒内完成收尾并上屏。
5. **交互本地化**：托盘菜单与关键状态文案全面中文化。

## 3. 问题-方案映射

| 用户问题 | 初步根因判断 | 0.6 对策 |
|---|---|---|
| 菜单为英文 | UI 文案未本地化 | 托盘菜单、状态文案、错误提示统一中文化（保留英文错误码） |
| 停顿时字幕消失 | partial 流在停顿后重置，Overlay 仅显示最新 partial | 新增“字幕累计显示模型”：稳定文本 + 当前片段双层显示，不因 partial 清空而丢历史 |
| 松键后只上屏前几个字 | 会话尾包晚到，存在跨会话事件污染/过早上屏 | 会话令牌隔离（session token）+ 收尾稳定屏障（quiet window）+ 上屏前最终快照冻结 |
| 无输入焦点提示 | 缺少可编辑焦点检测 | 增加可输入目标探测（焦点检查）并在 Overlay/托盘实时提示 |
| 半段先上屏，剩余下次出现 | Final 事件与 stop/paste 时序竞态 | 单会话事件栅栏：仅接收当前会话事件；旧会话尾包直接丢弃并记录日志 |
| 收尾过慢 | drain timeout 偏保守，缺少快速路径 | 自适应收尾：短句快收尾，长句稳收尾；达到“稳定文本阈值”走快速上屏 |

## 4. 版本范围与优先级

### P0（必须完成）
1. **会话事件隔离与上屏一致性**
   - 每次 `start_session()` 生成 `session_token`。
   - `Recognizer` 回调绑定 token；非当前 token 事件直接丢弃并打点 `stale_event_drop_count`。
   - 松键后上屏文本使用“本次会话冻结快照”，禁止被后续迟到事件覆盖。
2. **字幕持续显示机制**
   - Overlay 改为：`stable_text`（已稳定）+ `live_partial`（实时片段）。
   - 停顿期间保留 `stable_text`，仅更新 `live_partial` 状态提示（如“思考中...”），不清空正文。
3. **可输入焦点探测**
   - 会话开始时检测当前焦点是否可编辑。
   - 非可编辑目标时，Overlay 明确提示“当前未聚焦输入框（可继续识别，结果保留剪贴板）”。
4. **收尾时延优化**
   - 新增 `fast_finalize_path`：若最近 500ms 内无新事件且已有稳定文本，提前结束 drain。
   - 默认收尾策略改为“软超时 + 兜底上屏”，目标 P95 收尾 < 1.5s。
5. **中文菜单与文案统一**
   - 托盘菜单项、处理中状态、诊断导出结果统一中文。

### P1（建议完成）
1. 上屏确认机制（可配置）：上屏后短暂校验目标应用是否收到文本，失败时自动补一次粘贴。
2. 焦点状态彩色指示（绿=可输入，黄=未知，红=不可输入）。
3. 术语文案规范：对外统一“收尾中/重连中/可重试”等表达。

## 5. 关键设计方案

## 5.1 会话令牌隔离（防跨会话串流）
建议逻辑：
1. `start_session()` 递增 `session_id` 并生成 `session_token`。
2. recognizer 回调改为 `on_event(event, token)` 或闭包绑定 token。
3. `SessionController` 处理事件时校验 token，不匹配则忽略并记录健康事件。

预期收益：修复“本次只上前半段，后半段跑到下一次”的核心时序问题。

## 5.2 字幕累计显示模型（防停顿中断）
建议数据结构：
1. `display_stable_text`：已稳定内容（不会因停顿清空）。
2. `display_live_partial`：当前流式片段（可替换）。
3. `overlay_text = stable + live`。

显示策略：
1. 若 partial 重置/变短，不清空 `stable`。
2. 停顿超过阈值时显示轻量状态标记（例如“（思考中）”）。
3. Final 到达后合并进入 `stable`。

## 5.3 自适应收尾策略（降时延）
1. 基础 drain 超时保留（兜底安全）。
2. 增加快速收尾条件：
   - 已有可上屏文本；
   - 最近一段时间无新识别事件；
   - 无重连中状态。
3. 触发快速收尾后直接进入 `PASTING`，并异步记录诊断指标。

## 5.4 焦点可编辑检测
1. 启动识别时探测前台应用与可编辑控件状态（可用 AX API/焦点信息近似判断）。
2. 输出三态：`editable` / `unknown` / `not_editable`。
3. 在 Overlay 和托盘 tooltip 同步提示，避免“讲完才发现无焦点”。

## 6. 验收标准（0.6新增）
1. **字幕连续性**：
   - 用例：说话中停顿 2-3 秒后继续。
   - 标准：字幕不清空，历史内容持续可见（30 次重复通过）。
2. **上屏一致性**：
   - 用例：长句+停顿+继续，松键一次上屏。
   - 标准：一次会话一次完整输出，不出现“尾段到下次会话”现象（100 次通过率 >= 99%）。
3. **焦点可感知**：
   - 用例：输入框/桌面/不可编辑区域切换。
   - 标准：状态提示与真实可输入性一致率 >= 95%。
4. **收尾时延**：
   - 指标：`finalize_latency_ms`。
   - 标准：P95 < 1500ms，P99 < 2500ms。
5. **菜单本地化**：
   - 标准：托盘主菜单中文覆盖率 100%。

## 7. 测试策略增补
1. 新增“跨会话尾包污染”回归用例（模拟旧会话迟到 Final）。
2. 新增“partial 回退/重置”字幕连续性用例。
3. 新增“焦点不可编辑”提示准确性 Manual 用例。
4. 新增“快速收尾路径”性能回归（统计 p95）。

## 8. 里程碑
1. **M1（第1周）**：会话令牌隔离 + 字幕累计显示原型 + 单测。
2. **M2（第2周）**：焦点探测 + 自适应收尾 + 集成测试。
3. **M3（第3周）**：性能压测、实机灰度、文案中文化收尾。

## 9. 发布策略
1. 增加功能开关：`session_token_guard`、`fast_finalize_path`、`focus_hint`。
2. 先灰度 20%，重点观察：
   - `stale_event_drop_count`
   - `session_success_rate`
   - `avg_finalize_latency_ms`
   - `missing_text_report_rate`
3. 指标稳定 3 天后全量。

---

结论：0.6 以“**会话一致性 + 过程可见性 + 收尾速度**”为主线，优先修复你在 0.5 实际使用中暴露的体验断点，再考虑附加功能扩展。
