# 0.6版开发任务拆解：会话一致性与过程可见性

## 1. 目标与范围

版本目标：
1. 修复“本次上屏不完整、尾段跑到下一次会话”的一致性问题。
2. 修复“停顿 2-3 秒字幕消失”的过程中断问题。
3. 增加“是否聚焦可输入框”的实时提示。
4. 缩短收尾时延，改善松键后等待体验。
5. 完成菜单与关键文案中文化。

   - 增加 `session_token` 生命周期、事件校验、快收尾决策。

5. `main.py`
   - 统一中文菜单与状态提示；展示焦点状态和收尾状态。

## 5. 每项任务完成定义（DoD）

每个任务必须同时满足：
1. 有对应单元测试或集成测试。
2. 关键路径具备结构化日志字段（至少含 `session_id`、`session_token`、`state`、`error_code`）。
3. 任意异常后可回到 `IDLE` 且可继续下一次会话。
4. `tail -f ~/Library/Logs/ASR-Assistant.log` 可看到关键状态与健康指标。

## 7. 风险与应对

1. **AX 焦点探测受系统权限限制**：
   - 应对：输出 `unknown` 状态并保守提示，不阻断识别主流程。
2. **令牌隔离改动影响回调链路**：
   - 应对：先在适配层做兼容包装，避免一次性改所有调用点。
3. **快收尾导致提前截断**：
   - 应对：快收尾只在“有稳定文本 + quiet window 满足”时触发，并保留兜底路径。

## 8. 发布前检查单

1. `session_token_guard` 开关默认开启并可回退。
2. “停顿 2-3 秒”场景字幕连续显示通过 30 次回归。
3. “一次松键一次完整上屏”通过率 >= 99%。
4. P95 收尾时延 < 1.5s，P99 < 2.5s。
5. 中文菜单覆盖 100%，诊断导出可正常生成并包含 0.6 字段。
