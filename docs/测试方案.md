## 一、术语表

| 术语 | 定义 |
|---|---|
| Unit | 对单模块进行纯逻辑测试，外部依赖全部 Mock。 |
| Integration | 多模块协同测试，允许进程内真实调用，外部服务可替身。 |
| E2E | 从热键触发到上屏完成的端到端测试。 |
| Manual | 需要人工操作或系统权限配合的验证项。 |
| Soak | 长时间高频连续执行的稳定性压测。 |

## 四、测试分层设计

## 4.1 Unit（每次提交必跑）

覆盖模块：
1. `SessionController` 状态机转换、会话令牌校验、幂等行为。
2. `HotkeyAdapter` 防抖与重复触发过滤。
3. `Recorder` 的队列投递、Sentinel 注入、超时停录逻辑（Mock 麦克风）。
4. `RecognizerAdapter` 事件标准化、drain 收尾、错误映射、健康探针。
5. `TranscriptAggregator` 的 segment 合并与 final/partial 回退策略。
6. `OverlayView` 字幕累计显示模型（stable + live partial）。
7. `PasteService` 文本剪贴板保存/恢复与失败返回。
8. `ConfigStore` 读写、损坏恢复、默认值回退。
9. `Diagnostics` 健康日志解析、汇总、快照导出字段。

关键断言：
1. 状态机路径 `IDLE->RECORDING->FINALIZING_GRACE->RECOGNIZING_DRAIN->PASTING->IDLE` 全覆盖。
2. 旧会话 token 事件不会污染当前会话。
3. `stop_session()` 多次调用不抛异常。
4. `FinalResult` 缺失时可回退到 partial 缓存文本。

---

## 4.2 Integration（PR 必跑）

覆盖场景：
1. 录音队列 + 识别适配层联调（使用本地 wav 回放而非真实麦克风）。
2. `SessionController` + `RecognizerAdapter` + `TranscriptAggregator` + `PasteService` 联调。
3. 自愈流程联调：超时触发软重连，连续失败触发硬重置。
4. 会话令牌隔离联调：模拟旧会话迟到 Final 事件。
5. UI 事件总线联调：字幕持续显示与焦点状态提示可驱动 `OverlayView` 更新。

关键断言：
1. PartialResult 能多次更新且 FinalResult 收尾后上屏一次。
2. 旧会话迟到事件被丢弃且有计数日志。
3. 上屏失败时文本保留并返回 `NO_ACTIVE_TARGET`。
4. 恢复链路执行期间无重入和状态抖动。

---

## 4.3 E2E（主干每日定时）

执行条件：macOS Runner + 专用测试账号。

覆盖场景：
1. 短按流程：开始后快速结束，仍能安全收尾。
2. 长按流程：连续录音 5-10 秒并成功上屏。
3. 长停顿流程：说话-停顿 2-3 秒-继续说，最终整段上屏。
4. 跨会话污染流程：模拟迟到事件，不应进入下一次会话上屏。
5. 焦点切换流程：输入框与桌面切换，提示状态正确。
6. 断网流程：进行中断网，返回 `NETWORK_ERROR` 并可恢复。
7. 错误 API Key：返回 `AUTH_FAILED`，不崩溃。

说明：
1. E2E 使用受控测试窗口作为输入目标，避免随机焦点导致误报。
2. 真实云端调用设置每日限流预算。

---

## 4.4 Soak（主干每周定时）

执行条件：macOS Runner 或专用稳定性测试机。

覆盖场景：
1. 连续 200 次短按 + 50 次长按。
2. 每轮插入随机网络抖动，验证恢复能力。
3. 每 10 轮插入“停顿 2-3 秒”场景，验证字幕连续与收尾时延。

关键断言：
1. 会话成功率 >= 99%。
2. 无“必须重启应用”案例。
3. P95 `finalize_latency_ms` < 1500ms。
4. 失败日志包含 `session_id`、`session_token`、`state`、`error_code`、最近事件快照。

---

## 4.5 Manual（发布前人工验证）

覆盖场景：
1. 麦克风权限拒绝与恢复。
2. 辅助功能权限拒绝与恢复。
3. 两个目标应用中的实机上屏体验（如浏览器、编辑器）。
4. 输入法切换状态下的 Cmd+V 粘贴可用性。
5. 状态可感知：`录音中/收尾中/重连中/失败可重试` 文案一致性。
6. 菜单中文化与导出诊断入口可用性验证。
7. 焦点不可编辑提示准确性验证。

## 六、CI/CD 执行策略

## 6.1 CI 必跑集合（push / pull_request）

1. `pytest -m "unit or integration" --maxfail=1`
2. 覆盖率门禁：核心逻辑 >= 85%。
3. 状态机分支覆盖检查：`SessionController` 必须 100%。

## 6.2 主干定时集合（daily）

1. `pytest -m e2e`
2. 运行在 macOS runner。
3. 使用测试 API Key 与限流预算。

## 6.3 主干定时集合（weekly）

1. `pytest -m soak -v`
2. 执行稳定性压测与恢复能力验证。

## 6.4 本地发布前集合

1. `pytest -m "unit or integration or e2e"`
2. 执行一次 `pytest -m soak -k TC-05-SOAK-01`
3. 执行一次 `pytest -k "TC-06"`
4. 按 `Manual` 清单逐项人工勾验。

## 七、示例工作流（可直接作为基础模板）

```yaml
name: macOS ASR Tests
on:
  push:
  pull_request:
  schedule:
    - cron: '0 2 * * *'
    - cron: '0 3 * * 1'

jobs:
  ci-fast:
    if: github.event_name != 'schedule'
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install -r requirements.txt
      - run: pip install pytest pytest-cov pytest-qt pytest-asyncio
      - run: pytest -m "unit or integration" --cov=. --cov-fail-under=85

  e2e-daily:
    if: github.event.schedule == '0 2 * * *'
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install -r requirements.txt
      - run: pip install pytest pytest-qt pytest-asyncio
      - run: pytest -m e2e -v

  soak-weekly:
    if: github.event.schedule == '0 3 * * 1'
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install -r requirements.txt
      - run: pip install pytest pytest-qt pytest-asyncio
      - run: pytest -m soak -v
```
